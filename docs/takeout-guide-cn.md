# 申请YouTube Data API密钥（可选）

> ⚠️ **注意**：此步骤为**可选**。如果您只想使用导出的数据（观看历史HTML），可以跳过此步骤。
> 
> 如果您想获取视频的详细标签、分类、频道等信息，则需要申请API密钥。

---

### 为什么需要API？

| 数据来源 | 能获取的内容 | 是否需要API |
|----------|--------------|-------------|
| 观看记录HTML | 视频URL、观看时间 | ❌ 不需要 |
| YouTube API | 视频标题、描述、标签、分类、频道信息 | ✅ 需要 |

---

### 第一步：创建Google Cloud项目

1. 访问 [Google Cloud Console](https://console.cloud.google.com/)

2. 点击页面顶部的 **"选择项目"** → **"新建项目"**

3. 输入项目名称：
   ```
   YouTube Data Analyzer
   ```

4. 点击 **"创建"**

---

### 第二步：启用YouTube Data API v3

1. 在左侧菜单中点击 **"API和服务"** → **"库"**

2. 在搜索框中输入 **"YouTube Data API v3"**

3. 点击搜索结果中的 **YouTube Data API v3**

4. 点击 **"启用"**

---

### 第三步：创建API密钥

1. 在左侧菜单中点击 **"API和服务"** → **"凭据"**

2. 点击 **"+ 创建凭据"** → **"API密钥"**

3. 系统会自动创建一个API密钥，格式类似：
   ```
   AIzaSyCxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

4. **复制密钥并妥善保存！**

---

### 第四步：配置到项目中

1. 打开项目中的 `config.json` 文件

2. 将API密钥填入：
   ```json
   {
       "api": {
           "key": "在这里粘贴您的API密钥",
           "proxy": {
               "http": "http://proxy-dku.oit.duke.edu:3128",
               "https": "http://proxy-dku.oit.duke.edu:3128"
           }
       },
       "paths": {
           "takeout_dir": "您的Takeout文件夹路径"
       }
   }
   ```

---

### API配额限制

| 项目 | 限制 |
|------|------|
| 免费每日配额 | 10,000 单位 |
| videos.list() | 1 单位/请求 |
| 每日可查询视频数 | ~10,000 个 |

```
💡 提示：获取1000个视频详情 = 消耗1000单位
```

# 导出YouTube数据

## 准备工作

- 一个Google账号
- 稳定的网络连接
- 预计等待时间：30分钟 - 2小时

---

## 第一步：访问Google Takeout

### 1.1 打开Takeout页面

在浏览器中访问：

```
https://takeout.google.com
```

### 1.2 登录Google账号

如果您尚未登录，系统会要求您登录Google账号。

```
⚠️ 提示：请确保登录的是您主要的YouTube账号
```

---

## 第二步：选择要导出的数据

### 2.1 点击"选择要包含的服务"

页面会显示所有可导出的Google服务列表。

### 2.2 找到并勾选YouTube

在列表中找到 **YouTube 和 YouTube Music**，并勾选它。

```
☑️ YouTube 和 YouTube Music
```

### 2.3 点击"所有Google服务的复选框"展开选项

点击后会出现更多选项。

---

## 第三步：配置YouTube导出选项

### 3.1 选择要导出的数据类型

点击 **"多个格式"** 或 **"下一个"**，然后选择：

| 数据类型 | 说明 | 推荐 |
|----------|------|------|
| 观看历史 | 您的观看记录 | ✅ 必须 |
| 搜索记录 | 您的搜索历史 | ❌ 可选 |
| 评论 | 您的评论 | ❌ 可选 |

### 3.2 选择导出格式

| 格式 | 优点 | 推荐 |
|------|------|------|
| **HTML** | 易于解析，包含完整信息 | ✅ 推荐 |
| JSON | 便于程序处理 | ⭐ 备选 |

```
推荐选择：HTML格式
原因：HTML格式包含更丰富的信息，便于后续分类处理
```

---

## 第四步：设置传输方式

### 4.1 选择传输方式

| 方式 | 说明 | 推荐 |
|------|------|------|
| **发送到您的邮箱** | 下载链接发送到邮箱 | ✅ 推荐 |
| 直接下载 | 直接下载文件 | ❌ 不推荐（大文件） |
| 添加到Drive | 保存到Google Drive | ⭐ 备选 |

### 4.2 设置文件大小限制

如果选择"发送到您的邮箱"，可以设置分卷大小：

```
推荐：2 GB 或更大
原因：避免文件被分割成太多小块
```

---

## 第五步：创建导出

### 5.1 点击"创建导出"

```
⚠️ 注意：此过程可能需要30分钟到几小时不等
```

### 5.2 等待处理

页面会显示处理进度。

---

## 第六步：下载数据

### 6.1 收到邮件通知

处理完成后，您会收到一封来自Google的邮件。

### 6.2 点击下载链接

打开邮件，点击下载链接。

```
⚠️ 链接有效期：7天
⚠️ 请尽快下载
```

### 6.3 保存文件

将下载的文件保存到指定位置。

---

## 第七步：准备上传

### 7.1 解压文件（如果需要）

下载的文件可能是 `.zip` 格式，需要解压。

### 7.2 找到观看记录文件

解压后，文件夹结构如下：

```
Google_Takeout/
└── YouTube 和 YouTube Music/
    └── 历史记录/
        ├── 观看记录.html  ← 主要文件！
        └── 搜索记录.html
```

---

## 常见问题

### Q1: 导出需要多长时间？

A: 通常需要30分钟到2小时不等，取决于您的数据量。

### Q2: 链接过期了怎么办？

A: 重新访问 https://takeout.google.com ，点击"创建新的导出"即可。

### Q3: 我的数据安全吗？

A: 您的数据保存在您自己的设备上，我们不会上传或存储您的原始数据。

### Q4: 可以只导出观看历史吗？

A: 可以，在选择数据类型时只勾选"观看历史"即可。

### Q5: 文件太大无法处理怎么办？

A: 您可以设置更小的分卷大小（如500MB），文件会被分割成多个小文件。
